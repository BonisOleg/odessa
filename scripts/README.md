# Скрипти для захисту Django шаблонів від розривів тегів

## Проблема

Розриви Django тегів `{{ }}` та `{% %}` на кілька рядків призводять до:
- Небажаних пробілів у згенерованому HTML
- Проблем з форматуванням та відображенням
- Складності читання та підтримки коду

## Рішення

### 1. Автоматична перевірка

```bash
./scripts/check_template_tags.sh
```

Перевіряє всі HTML шаблони на наявність розривів тегів. Виходить з кодом помилки, якщо знайдено розриви.

### 2. Автоматичне виправлення

```bash
./scripts/fix_template_tags.sh
```

Автоматично виправляє розриви тегів у шаблонах (якщо можливо).

### 3. Git Pre-commit Hook

Налаштуйте git hook для автоматичної перевірки при кожному коміті:

```bash
./scripts/setup-git-hooks.sh
```

Після налаштування:
- При кожному коміті автоматично виправляються розриви (якщо можливо)
- Коміт блокується, якщо знайдено розриви, які не можна виправити автоматично

## Правила кодування

**КРИТИЧНО:** Ніколи не розривайте Django теги на кілька рядків!

### ❌ Погано:
```django
{{ company.name
    }}

{% if condition and
    other_condition %}
```

### ✅ Добре:
```django
{{ company.name }}

{% if condition and other_condition %}
```

### ✅ Альтернатива для довгих рядків:
```django
{% with name=company.name %}
    {{ name }}
{% endwith %}
```

## EditorConfig

Файл `.editorconfig` налаштований з `max_line_length = off` для HTML файлів, що дозволяє довгі рядки без автоматичного розриву тегів.

## Cursor AI Rules

Правила додано до `.cursorrules` - Cursor AI тепер знає, що не можна розривати теги.
